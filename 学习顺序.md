## 第 1 步：`infer.py` —— 最简单的入口（20 分钟）

📂 **[infer.py](infer.py)**（约30行）

### 为什么先看这个？

这是整个项目**最短、最完整的文件**。它展示了从输入到输出的完整流程，但没有深入模型细节。

### 阅读要点

```
加载模型 → 读音频(16kHz) → STFT → 模型推理 → iSTFT → 保存增强音频
```

逐行读完后，重点理解这几行：

| 行号 | 代码 | 理解要点 |
|------|------|---------|
| 18 | `torch.stft(..., 512, 256, 512, ...)` | STFT 三个关键参数：nfft、hop、win |
| 19 | `torch.view_as_real(input)` | 复数 → `(F, T, 2)`，最后一维是 [实部, 虚部] |
| 21 | `model(input[None])` | `[None]` = 添加 batch 维度 `(1, F, T, 2)` |
| 22 | `torch.view_as_complex(...)` | `(F, T, 2)` → 复数，为 iSTFT 准备 |
| 23 | `torch.istft(...)` | 频域 → 时域，参数必须和 STFT 一致 |

### ✅ 学完检查

能用自己的话解释：`mix` 是怎么一步步变成 `enh` 的？

## 第 2 步：`loss.py` —— 理解"什么是好的增强"（20 分钟）

📂 **[loss.py](loss.py)**（约40行）

### 为什么第二个看这个？

损失函数比模型简单得多，但能帮你理解：
- 模型输出的是什么（复数频谱，包含实部和虚部）
- "增强效果好"在数学上是怎么定义的

### 阅读要点

三种损失的组合：

| 损失 | 行号 | 公式 | 通俗理解 |
|------|------|------|---------|
| 幅度损失 | 22 | `MSE(mag^0.3)` | 响度分布要对 |
| 压缩复数损失 | 20-21 | `MSE(real/mag^0.7)` | 相位也要对（取0.7次方压缩范围） |
| SI-SNR | 27 | `-10*log10(signal/noise)` | 整体信噪比越高越好 |

关键设计：**幂次压缩**（`**0.3`、`**(0.7)`）防止大声音主导损失函数。

### ✅ 学完检查

为什么不能只优化幅度，还要优化实部和虚部？

## 第 3 步：`gtcrn.py` —— 核心模型（2-3 小时，分 3 次读）

📂 **[gtcrn.py](gtcrn.py)**（约340行）

这是最重要的一步。**不要从头到尾读**，按下面顺序分 11 个子模块逐个理解。

### 阅读路线图

```
第一阶段：基础积木（简单，30分钟）
  ├─ 3.1 ConvBlock   → 理解 "Conv + BN + 激活" 的标准写法
  ├─ 3.2 SFE         → 理解 nn.Unfold 滑动窗口展开
  └─ 3.3 Mask        → 理解复数乘法 (a+bi)×(c+di)

第二阶段：中级模块（核心组件，45分钟）
  ├─ 3.4 ERB         → 理解人耳频率尺度 + 三角滤波器组
  ├─ 3.5 TRA         → 理解 GRU 生成注意力权重
  ├─ 3.6 GRNN        → 理解 "分成两组各自跑RNN"
  └─ 3.7 Encoder     → 理解 5 层编码器的膨胀率设计

第三阶段：核心创新（重点！60分钟）
  ├─ 3.8 GTConvBlock → ⭐ ShuffleNetV2 + 膨胀卷积 + TRA 融合
  ├─ 3.9 DPGRNN      → ⭐ 双路径 RNN（频率+时间两个维度建模）
  └─ 3.10 Decoder    → 与 Encoder 对称 + 跳跃连接

第四阶段：组装（15分钟）
  └─ 3.11 GTCRN      → 回头看全局数据流，检查是否理解每一步 shape 变化
```

### 3.1 ConvBlock（第 96-104 行）⭐⭐

```
Conv2d → BatchNorm → PReLU（或Tanh）
```

- `groups` 参数的作用：分组卷积，减少参数
- `use_deconv`：控制用普通卷积还是转置卷积
- `is_last`：最后一层用 Tanh（输出 `[-1,1]`）

### 3.2 SFE（第 64-74 行）⭐⭐

```
Unfold(kernel=1×3) → reshape → (B, C×3, T, F)
```

- 在频率维度上做 1×3 滑动窗口
- 3通道变9通道：每个位置看到自己和左右邻居
- 类比：图像处理中的"patch extraction"

### 3.3 Mask（第 265-274 行）⭐⭐

```
s_real = M_r*S_r - M_i*S_i
s_imag = M_i*S_r + M_r*S_i
```

- 就是复数乘法：`(M_r + jM_i) × (S_r + jS_i)`
- 为什么用复数掩码？—— 同时修正幅度和相位

### 3.4 ERB（第 11-61 行）⭐⭐⭐

```
频率(Hz) ↔ ERB刻度 互相转换 → 三角滤波器组 → 压缩/恢复
```

- **ERB**：等效矩形带宽，人耳对频率的非线性感知
- **hz2erb / erb2hz**：频率 ↔ ERB 刻度公式
- **erb_filter_banks**：在ERB刻度上均匀采样 → 三角滤波器 → 映射回线性频率
- **bm（压缩）**：257 频带 → 65低频直通 + 64 ERB高频 = 129
- **bs（恢复）**：129 → 257

### 3.5 TRA（第 77-93 行）⭐⭐⭐

```
能量计算 → GRU建模时间依赖 → Sigmoid生成注意力 → 加权
```

- `zt = mean(x²)`：每帧的能量
- GRU：学习能量变化的时间模式
- 注意力：重要的帧放大，不重要的抑制
- **为什么用GRU？**—— 计算量小，适合实时处理

### 3.6 GRNN（第 156-183 行）⭐⭐⭐

```
输入分两组 → 各自独立GRU → 输出拼接
```

- 分组的好处：参数量约为不分组的 1/2
- 两组之间没有交互 —— 留给 Channel Shuffle 来做

### 3.7 Encoder（第 228-244 行）⭐⭐

```
ConvBlock(下采样频率) × 2  →  GTConvBlock(扩张感受野) × 3
```

| 层 | 膨胀率 | 时间感受野 | 作用 |
|----|--------|-----------|------|
| 1-2 | — | 局部 | 压缩频率（129→65→33） |
| 3 | dil=1 | 3帧 | 短时模式（音素级别） |
| 4 | dil=2 | 7帧 | 中时模式（音节级别） |
| 5 | dil=5 | 15帧 | 长时模式（词级别） |

> 💡 膨胀率递增 = 从局部到全局的多尺度感受野

### 3.8 GTConvBlock（第 107-153 行）⭐⭐⭐⭐

**GTCRN 最核心的创新模块**，务必仔细读！

```
输入16通道
  ├─ x1(前8通道): SFE → PointConv → DepthConv(膨胀) → PointConv → TRA
  ├─ x2(后8通道): 直接跳过（不处理）
  └─ shuffle(x1, x2) → 交叉合并 → 输出16通道
```

四个关键设计：

| 设计 | 目的 |
|------|------|
| **Channel Split** | 一半通道不处理，省计算（ShuffleNetV2） |
| **Depthwise Conv** | 每个通道独立卷积，参数极少（分组数=通道数） |
| **膨胀卷积** | 不增加参数但扩大时间感受野 |
| **Channel Shuffle** | 处理后和未处理的通道交叉混合，信息流通 |

### 3.9 DPGRNN（第 186-225 行）⭐⭐⭐⭐

**双路径RNN** —— 语音增强领域的经典架构：

```
输入 (B,C,T,F)
  │
  ├─ Intra-RNN ────────────────────
  │   (B,T,F,C) → (B*T,F,C)        # 把B和T合并
  │   双向GRU 沿频率(F)维度建模      # 同一时刻不同频率的关系
  │   + 残差连接                    # 防止信息丢失
  │
  └─ Inter-RNN ────────────────────
      (B,F,T,C) → (B*F,T,C)        # 把B和F合并
      单向GRU 沿时间(T)维度建模      # 同一频率随时间的变化
      + 残差连接                    # 因果！只看过去
```

| | Intra-RNN | Inter-RNN |
|------|-----------|-----------|
| 沿什么维度？ | 频率 (F) | 时间 (T) |
| 双向还是单向？ | 双向（频率无因果约束）| 单向（时间有因果约束）|
| 建模什么？ | 谐波结构（基频+泛音关系）| 时序变化（语音的开始/结束） |

### 3.10 Decoder（第 247-262 行）⭐⭐

```
GTConvBlock(反卷积) × 3  →  ConvBlock(上采样频率) × 2
```

- 与 Encoder 对称，膨胀率从 5→2→1（大→小）
- **跳跃连接**：`x + en_outs[4-i]` —— 解码器每层接收编码器对应层的输出
- 为什么用跳跃连接？—— 保留高频细节，类似 U-Net

### 3.11 GTCRN（第 277-318 行）⭐⭐⭐

**回头看全局**，画出完整数据流：

```
spec (B, 257, T, 2)
  → 提取 mag+real+imag → (B, 3, T, 257)
  → ERB.bm → (B, 3, T, 129)     # -128频带
  → SFE → (B, 9, T, 129)         # ×3通道
  → Encoder → (B, 16, T, 33)     # 压缩频率+时序建模
  → DPGRNN×2 → (B, 16, T, 33)    # 时频联合建模
  → Decoder → (B, 2, T, 129)     # 恢复频率+跳跃连接
  → ERB.bs → (B, 2, T, 257)      # +128频带
  → Mask(复数乘法) → (B, 257, T, 2)
```

## 第 4 步：流式推理 —— 理解"实时处理"（1.5 小时）

离线模型一次处理整句话，但实时场景需要**逐帧处理**。

### 4.1 流式卷积：`stream/modules/convolution.py`（30分钟）

📂 **[convolution.py](stream/modules/convolution.py)**（约330行）

核心思想：**用缓存替代时间维度的 padding**

```
非流式:  pad([0,0, kT-1, 0]) → Conv2d(全部帧)
流式:    cat(cache, 新1帧) → Conv2d → 更新cache
```

三个类按顺序读：
1. **StreamConv1d**（16-48行）：一维版本，最直观
2. **StreamConv2d**（51-93行）：时频二维，时间缓存+频率 padding
3. **StreamConvTranspose2d**（169-262行）：v2版本，用 Conv2d 实现反卷积

### 4.2 权重转换：`stream/modules/convert.py`（15分钟）

📂 **[convert.py](stream/modules/convert.py)**（约40行）

- 将离线模型的预训练权重复制到流式模型
- 特殊处理：转置卷积的权重需要**时间维度翻转**（因为用 Conv2d 模拟）
- 读完后能回答：为什么能复用离线模型的权重？

### 4.3 流式模型：`stream/gtcrn.py`（30分钟）

📂 **[gtcrn.py](stream/gtcrn.py)**（约350行）

对照离线版 `gtcrn.py`，找三个关键差异：

| 模块 | 离线版 | 流式版 | 关键变化 |
|------|--------|--------|---------|
| **GTConvBlock** | `GTConvBlock` | `StreamGTConvBlock` | forward 多了 `conv_cache` 和 `tra_cache` 参数 |
| **TRA** | `TRA` | `StreamTRA` | GRU 的隐藏状态 `h_cache` 在帧间传递 |
| **DPGRNN** | `DPGRNN` | `DPGRNN` | Inter-RNN 多了 `inter_cache` 参数 |

### 4.4 完整流式推理：`stream/gtcrn_stream.py`（20分钟）

📂 **[gtcrn_stream.py](stream/gtcrn_stream.py)**（约135行）

- 对比离线推理和逐帧推理的结果误差（应接近0）
- ONNX 导出：PyTorch → ONNX → ONNX Runtime 推理
- 理解 RTF（Real-Time Factor）：推理时间 / 音频时长

## 第 5 步：LADSPA 插件 —— 部署到真实系统（选读，1-2 小时）

> 📂 `ladspa/` 目录是 Rust 编写的 Linux 音频插件，用于通过 PipeWire/JACK 在系统级拦截和处理音频。

### 5.1 了解架构

| 文件 | 作用 | 对应 Python 的哪个部分 |
|------|------|----------------------|
| [ladspa/src/stft.rs](ladspa/src/stft.rs) | STFT/iSTFT 处理 | `torch.stft` / `torch.istft` |
| [ladspa/src/model.rs](ladspa/src/model.rs) | ONNX 模型加载 + 逐帧推理 | `GTCRN.forward()` |
| [ladspa/src/plugin.rs](ladspa/src/plugin.rs) | 双线程架构 + 重采样 | `stream/gtcrn_stream.py` 推理循环 |
| [ladspa/src/lib.rs](ladspa/src/lib.rs) | LADSPA 插件入口 + 端口定义 | — |

### 5.2 核心设计亮点

1. **双线程架构**：音频线程（实时）↔ 环形缓冲区 ↔ 工作线程（推理）
2. **无锁通信**：用原子变量代替互斥锁，避免音频卡顿
3. **高质量重采样**：48kHz（主机）↔ 16kHz（模型），用 FFT 加速的 sinc 插值
4. **预分配缓冲区**：初始化后零堆分配，满足实时要求

## 遇到看不懂的术语怎么办？

| 术语 | 简单解释 | 在哪看 |
|------|---------|--------|
| STFT | 把音频切成小段，每段做FFT，得到"频谱随时间变化"的图 | infer.py |
| 频谱图 | 横轴=时间，纵轴=频率，颜色=能量 | infer.py |
| 卷积 (Conv) | 用一个小的"模板"在数据上滑动，提取局部特征 | gtcrn.py ConvBlock |
| 膨胀卷积 | 模板元素之间有间隔，感受野更大但参数不变 | gtcrn.py Encoder |
| 转置卷积 | 卷积的"逆操作"，用于上采样/恢复分辨率 | gtcrn.py Decoder |
| GRU | 一种轻量的循环神经网络，适合处理序列 | gtcrn.py TRA |
| 批归一化 (BN) | 让每层输出的数值范围稳定，训练更稳定 | gtcrn.py ConvBlock |
| 残差连接 | 输出 = 输入 + 变换(输入)，防止信息丢失 | gtcrn.py DPGRNN |
| Channel Shuffle | 把不同组的通道交错混合，信息流通 | gtcrn.py GTConvBlock |
| CRM | 复数比率掩码：在频域直接修正幅度+相位 | gtcrn.py Mask |
| RTF | 实时因子：推理时间/音频时长，<1即满足实时 | stream/gtcrn_stream.py |
