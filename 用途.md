# GTCRN 项目用途

## 一句话总结

GTCRN（Grouped Temporal Convolutional Recurrent Network，分组时序卷积循环网络）是一个**超低计算资源的语音增强模型**，用于从带噪声的语音中提取干净的人声。

---

## 核心用途：语音增强（Speech Enhancement）

简单来说，这个项目的功能是：**输入一段包含噪声的语音，输出一段干净的语音**。

### 实际应用场景

| 场景 | 说明 |
|------|------|
| **通话降噪** | 在嘈杂环境中打电话时，实时滤除背景噪声（如风声、车流声、咖啡馆噪音） |
| **助听设备** | 帮助听障人士在噪声环境中听清对话 |
| **语音助手** | 作为语音识别（ASR）的前处理，提高 Alexa/Siri/小爱同学等在噪声环境下的识别准确率 |
| **在线会议** | Zoom/腾讯会议等实时降噪，让远程通话更清晰 |
| **直播/K歌** | 实时去除环境噪声，保留干净人声 |
| **嵌入式设备** | 由于模型极小（48.2K 参数），可以部署在耳机、智能音箱等资源受限的设备上 |

---

## 为什么这个项目特别？

### 1. 极致轻量
- 仅 **48.2K 参数**（相当于一张 200×200 像素的小图片大小）
- 每秒仅需 **33.0 MMACs**（百万次乘加运算）
- 比经典轻量模型 RNNoise 效果更好，但计算量更小

### 2. 实时流式处理
- `stream/` 目录提供了流式推理版本
- 在普通 CPU（i5-12400）上，实时因子（RTF）仅为 **0.07**
  - 意思是处理 1 秒的音频只需 0.07 秒，远快于实时要求（RTF < 1）

### 3. 两种工作模式

| 模式 | 文件 | 适用场景 |
|------|------|---------|
| **离线模式**（整句处理）| `gtcrn.py` + `infer.py` | 录音后处理、批量处理音频文件 |
| **流式模式**（逐帧处理）| `stream/` 目录 | 实时通话、直播、在线会议 |

### 4. 可部署到多种平台
- **PyTorch**：训练和研究
- **ONNX**：跨平台高性能推理（`stream/gtcrn_stream.py` 支持导出 ONNX）
- **LADSPA 插件**：Linux 上的 PipeWire 音频管线（`ladspa/` 目录，Rust 实现）
- **sherpa-onnx**：已被社区集成，支持更多平台
- **WebAssembly**：可在浏览器中运行

---

## 输入 → 输出

```
输入：16kHz 采样率的单声道带噪语音（.wav）
  │
  ▼
  STFT（短时傅里叶变换）→ 频谱图
  │
  ▼
  GTCRN 神经网络 → 估计干净语音的频谱掩码（Complex Ratio Mask）
  │
  ▼
  iSTFT（逆短时傅里叶变换）→ 增强后的语音波形
  │
  ▼
输出：16kHz 采样率的增强语音（.wav）
```
