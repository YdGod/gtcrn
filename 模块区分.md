## 一、`gtcrn.py` —— 核心模型文件（离线版本）

这是整个项目**最核心**的文件，定义了所有神经网络模块。按照从底层到顶层的顺序：

### 1.1 ERB（等效矩形带宽模块）—— 第 11-61 行

```
作用：模拟人耳听觉特性，压缩频率维度
```

- 人耳对低频声音的分辨能力强于高频，ERB 尺度模拟了这一特性
- **`hz2erb` / `erb2hz`**：频率 ↔ ERB 刻度的互相转换
- **`erb_filter_banks`**：构建三角滤波器组（类似梅尔滤波器组，但是按 ERB 尺度）
- **`bm`（band merging）**：将 257 个线性频带 → 129 个频带（低频保留 65 个 + 高频压缩为 64 个 ERB 频带）
- **`bs`（band splitting）**：逆操作，129 个频带 → 恢复为 257 个频带

> 📌 **关键参数**：`erb_subband_1=65`（低频直通数）, `erb_subband_2=64`（高频 ERB 压缩后频带数）

### 1.2 SFE（子带特征提取）—— 第 64-74 行

```
作用：将相邻频带的信息融合，扩展特征通道
```

- 在频率维度上做 1×3 的滑动窗口展开（Unfold）
- 输入 `(B, C, T, F)` → 输出 `(B, C×3, T, F)`，通道数变为 3 倍
- 例如：3 通道 → 9 通道（捕获每个频带的邻居信息）

### 1.3 TRA（时序循环注意力）—— 第 77-93 行

```
作用：让模型自动学习"哪些时间帧更重要"
```

- 对每个通道计算时间维度的能量 `zt = mean(x²)`
- 用 GRU 建模能量的时间变化规律
- 通过 Sigmoid 生成 0~1 的注意力权重
- 用权重对原始特征加权：重要的帧放大，不重要的帧抑制

### 1.4 ConvBlock（基础卷积块）—— 第 96-104 行

```
作用：最基础的卷积-归一化-激活单元
```

- 封装了 `Conv2d / ConvTranspose2d + BatchNorm2d + 激活函数`
- 最后一层用 Tanh（输出范围 [-1,1]），其他层用 PReLU
- 支持分组卷积（groups）和反卷积（use_deconv）

### 1.5 GTConvBlock（分组时序卷积块）⭐ 核心创新 —— 第 107-153 行

```
作用：ShuffleNetV2 风格的轻量卷积 + 时序注意力
```

这是 GTCRN 名字来源的核心模块，结构如下：

```
输入 x (16通道)
  │
  ├── x1 (前8通道)              x2 (后8通道) ──┐
  │     │                                      │
  │     ├─ SFE (3→24通道)                      │
  │     ├─ Point Conv (1×1)                    │
  │     ├─ Depth Conv (3×3, 时序膨胀卷积)      │
  │     ├─ Point Conv (1×1)                    │
  │     ├─ TRA (时序注意力)                    │
  │     │                                      │
  │     └── h1 ──── shuffle 交织 ──── x2 ──────┘
  │                              │
  └──────────────────────────────┘
                输出 (16通道)
```

**关键设计思想**：
- **Channel Split**：一半通道做变换，另一半直接跳过（减少计算量，类似 ShuffleNetV2）
- **Depthwise 膨胀卷积**：在时间维度上用膨胀率 1/2/5 扩展感受野（看到越来越长的历史信息）
- **Channel Shuffle**：将变换后的通道和原始通道交织，实现跨组信息交流
- **TRA 注意力**：让模型关注重要的时间点
- Encoder 和 Decoder 共用，通过 `use_deconv` 控制是普通卷积还是反卷积

### 1.6 GRNN（分组 RNN）—— 第 156-183 行

```
作用：将输入分成两组，分别用独立的 GRU 处理
```

- 输入按通道一分为二
- 每半分别经过独立的单层 GRU
- 输出拼接回去
- **好处**：减少参数量（分组后参数量约为原来的 1/2）

### 1.7 DPGRNN（分组双路径 RNN）⭐ 核心创新 —— 第 186-225 行

```
作用：既在频率维度建模（Intra-RNN），又在时间维度建模（Inter-RNN）
```

这是语音增强领域的经典模块，GTCRN 将其与分组策略结合：

```
输入 (B, C, T, F)
  │
  ├─ Intra-RNN ─────────────────────
  │   reshape → (B*T, F, C)         │
  │   GRNN (双向) 沿频率轴建模      │
  │   FC + LayerNorm + 残差连接     │
  │                                 │
  ├─ Inter-RNN ─────────────────────
  │   permute → (B, F, T, C)        │
  │   reshape → (B*F, T, C)         │
  │   GRNN (单向) 沿时间轴建模      │
  │   FC + LayerNorm + 残差连接     │
  │                                 │
  └── 输出 (B, C, T, F)
```

- **Intra-RNN**：沿频率轴建模（双向 GRU）—— 同一个时间帧内，各频带之间的关系
- **Inter-RNN**：沿时间轴建模（单向 GRU，保证因果性）—— 同一频带在不同时间的变化
- 两层 DPGRNN 串联（`dpgrnn1` → `dpgrnn2`），逐层精炼特征

### 1.8 Encoder（编码器）—— 第 228-244 行

```
作用：逐层压缩频率维度，提取高层次特征
```

5 层结构：
| 层 | 类型 | 频率维度变化 | 作用 |
|----|------|-------------|------|
| 1 | ConvBlock (1,5) stride=2 | 129 → 65 | 压缩频率 |
| 2 | ConvBlock (1,5) stride=2 | 65 → 33 | 压缩频率 |
| 3 | GTConvBlock dil=1 | 保持 33 | 时序建模（小感受野）|
| 4 | GTConvBlock dil=2 | 保持 33 | 时序建模（中感受野）|
| 5 | GTConvBlock dil=5 | 保持 33 | 时序建模（大感受野）|

- 前两层在**频率维度压缩**（stride=2），模仿 CNN 图像编码器在空间维度下采样
- 后三层在**时间维度扩展感受野**（膨胀率递增），捕获不同时间尺度的模式

### 1.9 Decoder（解码器）—— 第 247-262 行

```
作用：逐层恢复频率维度，并通过跳跃连接融合编码器特征
```

5 层结构，与 Encoder 对称但方向相反：
| 层 | 类型 | 频率维度变化 |
|----|------|-------------|
| 1 | GTConvBlock (反卷积) dil=5 | 保持 33 |
| 2 | GTConvBlock (反卷积) dil=2 | 保持 33 |
| 3 | GTConvBlock (反卷积) dil=1 | 保持 33 |
| 4 | ConvBlock (反卷积) stride=2 | 33 → 65 |
| 5 | ConvBlock (反卷积) stride=2 | 65 → 129 |

- 每层接收 Encoder 对应层的输出作为跳跃连接（`x + en_outs[4-i]`）
- 好处：保留细节信息，缓解梯度消失

### 1.10 Mask（复数比率掩码）—— 第 265-274 行

```
作用：将估计的掩码应用到原始频谱上
```

- 用复数乘法 `(a+bi) × (c+di)` 将掩码作用于原始频谱的实部和虚部
- 这是频域语音增强的标准做法

### 1.11 GTCRN（主模型）—— 第 277-318 行

```
作用：将所有模块组装成完整的语音增强模型
```

完整数据流：

```
原始音频波形
  │
  ▼ STFT
频谱 (B, F, T, 2)  [实部+虚部]
  │
  ├─ 提取特征：幅度谱 + 实部 + 虚部 → (B, 3, T, 257)
  │
  ├─ ERB.bm()：257 频带 → 129 频带
  │
  ├─ SFE：3 通道 → 9 通道
  │
  ├─ Encoder：129 → 33 (频率压缩)
  │
  ├─ DPGRNN × 2：双路径时序-频率联合建模
  │
  ├─ Decoder + skip connections：33 → 129 (频率恢复)
  │
  ├─ ERB.bs()：129 频带 → 257 频带
  │
  ├─ Mask：复数掩码作用于原始频谱
  │
  ▼ iSTFT
增强音频波形
```

## 二、`loss.py` —— 损失函数

### HybridLoss（混合损失）—— 第 5-30 行

```
作用：训练时告诉模型"什么是好的增强效果"
```

由三部分组成：

| 损失分量 | 公式 | 权重 | 作用 |
|----------|------|------|------|
| **幅度损失** | `MSE(mag^0.3_pred, mag^0.3_true)` | ×70 | 让增强后的声音能量分布接近干净语音 |
| **实部损失** | `MSE(real_compressed_pred, real_compressed_true)` | ×30 | 约束频谱的实部 |
| **虚部损失** | `MSE(imag_compressed_pred, imag_compressed_true)` | ×30 | 约束频谱的虚部 |
| **SI-SNR 损失** | `-10*log10(‖s_target‖² / ‖e_noise‖²)` | ×1 | 直接优化信号级别的信噪比 |

- 对幅度做 **0.3 次幂压缩**（`mag^0.3`）：压缩动态范围，让小信号和大信号被同等重视
- **SI-SNR**：尺度不变的信噪比，衡量增强语音与干净语音的相似度（越高越好）

## 三、`infer.py` —— 离线推理脚本

```
作用：用预训练模型处理一段音频，保存增强结果
```

流程极简：
1. 加载模型 → 加载预训练权重
2. 读取音频 (16kHz) → STFT → 模型推理 → iSTFT
3. 保存增强后的音频

## 四、`stream/` —— 流式推理模块

### 与离线版本的区别

| | 离线版本 | 流式版本 |
|------|---------|---------|
| 输入 | 完整音频，一次处理 | 逐帧输入，实时输出 |
| 时间建模 | 双向 RNN（看到未来）| 单向 GRU（只看历史）+ 状态缓存 |
| 注意力 | TRA（GRU 处理全部帧）| StreamTRA（逐帧更新 GRU 状态）|
| 卷积 | 普通 Conv2d | StreamConv2d（缓存历史帧）|

### 关键文件

- **`stream/gtcrn.py`**：流式模型定义（StreamGTCRN），结构与离线版一致但所有时序模块都改为流式版本
- **`stream/gtcrn_stream.py`**：流式推理 + ONNX 导出 + ONNX Runtime 推理
- **`stream/modules/convolution.py`**：实现带缓存的流式卷积
- **`stream/modules/convert.py`**：将离线模型权重转换为流式模型

## 五、`ladspa/` —— LADSPA 音频插件

```
作用：在 Linux 上作为系统级音频插件运行，实时处理所有音频输出
```

- 使用 **Rust** 编写，调用 ONNX Runtime 进行推理
- 通过 PipeWire 管线拦截和处理音频
- 社区贡献功能（实验性）
